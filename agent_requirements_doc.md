# Agent 平台功能升级与学习实操需求文档 (PRD)

## 1. 文档概述
本文件旨在定义 `AgentChat` 平台的进阶功能开发需求。通过对现有对话功能的升级，实现从简单的“模型对话”向“自主 Agent”架构的转型，并作为开发者的学习实操路径。

## 2. 目标用户
*   **初级 AI 开发者**：希望通过实操掌握 Agent 开发的核心概念。
*   **平台使用者**：需要能够自定义能力、具备专业背景知识的智能助手。

## 3. 功能需求详情

### 3.1 角色定制化 (Persona & System Prompt)
*   **背景**：目前创建 Agent 仅支持名称和模型选择，无法设定其行为模式。
*   **需求点**：
    *   **后端存储**：在 `AgentInfo` 实体中增加 `system_prompt` 字段。
    *   **Agent 实例化**：在构建 `ReActAgent` 时，必须注入 `system_prompt` 作为 LLM 的初始指令。
    *   **前端交互**：在“创建 Agent”弹窗中，提供多行文本框输入系统提示词，并支持预设模板（如：翻译官、代码专家、心理医生）。

### 3.2 工具集成与能力扩展 (Tool Calling)
*   **背景**：Agent 需要具备操作外部世界的能力。
*   **需求点**：
    *   **工具定义规范**：基于 AgentScope 框架，定义标准工具接口（如 `get_weather`, `web_search`）。
    *   **工具注册机制**：在创建 Agent 时，支持勾选或动态分配工具权限。
    *   **执行逻辑**：当 LLM 输出符合工具调用格式时，后端自动执行本地函数并返回 `Observation`（观察结果）给 Agent。

### 3.3 智能记忆管理 (Memory Management)
*   **背景**：长对话会导致 Token 溢出或响应变慢。
*   **需求点**：
    *   **滑动窗口记忆**：仅保留最近 N 轮对话作为上下文。
    *   **摘要持久化**：当对话轮数超过阈值，调用 LLM 对历史进行摘要总结，减少上下文长度。

### 3.4 推理过程可视化 (Reasoning Display)
*   **背景**：Agent 的思考过程（Thought）目前对用户不可见，无法判断其逻辑。
*   **需求点**：
    *   **流式输出解析**：前端需识别 Agent 输出中的 `Thought`, `Action`, `Observation` 等标签。
    *   **UI 展示**：在聊天界面中使用折叠面板或特殊样式展示 Agent 的“思考过程”，提升透明度。

## 4. 技术栈要求
*   **后端**：Java 17+, Spring Boot 3.x, AgentScope SDK
*   **前端**：Vue 3 (Vite), TypeScript, CSS3
*   **模型**：支持 OpenAI 协议的模型（如 DeepSeek, Qwen）

## 5. 验收标准
1.  **角色一致性**：创建特定角色的 Agent 后，其回答风格需严格符合 Prompt 设定。
2.  **工具有效性**：询问实时信息时，Agent 能够正确触发工具调用并给出准确回复。
3.  **流程完整性**：前端能够清晰展示 Agent 从接收问题、思考、执行工具到最终回复的完整链路。

---

## 6. 实施路线图 (Milestones)
*   **T1 (Phase 1)**: 完成基础角色定制与持久化。
*   **T2 (Phase 2)**: 实现第一个核心工具（如 Weather Tool）的闭环调用。
*   **T3 (Phase 3)**: 优化前端 UI，支持推理链路展示。
*   **T4 (Phase 4)**: 引入私有知识库 (RAG)，赋予 Agent 检索本地文档的能力。
*   **T5 (Phase 5)**: 实现人机协同 (Human-in-the-loop)，为高敏感工具（如数据库修改）增加人工审批流程。
*   **T6 (Phase 6)**: 探索多智能体 (Multi-Agent) 协作与工作流编排，解决复杂多步任务。

---

## 7. 进阶与高级需求 (Advanced Features)

### 7.1 检索增强生成 (RAG - Knowledge Base)
*   **背景**：Agent 需要结合企业的私有数据或最新的外部文档来回答问题，避免产生“幻觉”。
*   **需求点**：
    *   **文档解析与向量化**：支持上传文本/PDF文档，利用 Embedding 模型转换为向量并存储到向量数据库（如 Milvus / Chroma）。
    *   **检索与挂载**：在 Agent 执行过程中，先根据用户问题在向量库中进行语义检索 (Similarity Search)，将相关内容作为 Context 挂载到 Prompt 中再交给大模型。

### 7.2 多智能体协作 (Multi-Agent Collaboration)
*   **背景**：针对复杂的业务场景，单一的 Agent 往往难以兼顾所有逻辑，需要多个专职 Agent 协同工作。
*   **需求点**：
    *   **角色编排**：设计不同的 Agent（如 Coder Agent, Reviewer Agent, QA Agent）。
    *   **消息路由**：实现一个中控（Manager Agent）或者流水线机制，让不同的 Agent 能够互相传递消息、相互纠错。

### 7.3 高级规划与反思机制 (Planning & Reflection)
*   **背景**：基础的 ReAct 模式在面对长链路任务时容易迷失方向。
*   **需求点**：
    *   **任务拆解 (Plan-and-Solve)**：Agent 在行动前必须先输出一份 Step-by-step 的计划书。
    *   **自我纠错 (Self-Correction)**：在工具调用失败或结果不符合预期时，Agent 能够根据错误信息反思，并尝试采用替代方案而不是直接报错。

### 7.4 人机协同与安全审批 (Human-in-the-loop, HITL)
*   **背景**：Agent 操作外部系统的权限过大存在安全隐患，某些操作必须由人类确认。
*   **需求点**：
    *   **执行挂起**：当检测到敏感工具调用（如发送邮件、系统写操作）时，后端将状态机挂起，并推送审批卡片到前端。
    *   **人工干预**：人类点击“同意”、“拒绝”或“修改参数”后，唤醒后端状态机继续后续逻辑。

### 7.5 可控工作流引擎 (Workflow Engine)
*   **背景**：在生产环境中，完全由 LLM 驱动的 Agent 往往不够稳定，需要与传统的可控业务流结合。
*   **需求点**：
    *   **图结构执行**：通过有向无环图 (DAG)编排 Agent 的执行路径（类似 LangGraph）。
    *   **条件流转**：结合传统的 If/Else 逻辑和 LLM 的意图识别，使得流程既具备智能化又有确定的边界。

## 8. 开源工程化与快速启动 (Open Source & Quick Start)
*   **目标**：实现“5分钟上手”，降低部署门槛。
*   **需求点**：
    *   **Docker 一键部署**：提供 `docker-compose.yml`，包含后端、前端、数据库及向量库。
    *   **环境预检机制**：系统启动时自动检查数据库连接、API Key 有效性及必要的存储目录权限。
    *   **配置模版化**：提供详尽的 `.env.example` 与 `application.yml.template`。
    *   **演示模式 (Demo Mode)**：内置 Mock 数据和 Mock Model，允许用户在未配置 API Key 的情况下体验 UI 交互。

## 9. 资源库与内置插件 (Asset Library & Built-in Plugins)
*   **目标**：消除冷启动焦虑，提供开箱即用的能力。
*   **需求点**：
    *   **Agent 模版市场**：预置 10+ 高质量 Agent（如：代码审计员、周报生成器、学术翻译、数据库专家）。
    *   **常用工具集**：内置 Google Search (SerpApi)、Python 解释器（沙箱运行）、文件读取器、精准计算器。
    *   **Prompt 优化助手**：内置一个专门优化提示词的 Agent，帮助用户将模糊需求转化为专业的 System Prompt。

## 10. 交互体验深度优化 (UX Deep Optimization)
*   **目标**：打造 premium 级别的视觉与交互感受。
*   **需求点**：
    *   **增强渲染支持**：完美支持 Markdown、LaTeX 公式、Mermaid 流程图、代码高亮。
    *   **Token & 成本监控**：在对话侧边栏实时展示单次会话及累计的 Token 消耗与估算成本。
    *   **全平台适配**：支持精致的暗黑模式，适配移动端浏览器。
    *   **对话导出**：支持将会话导出为图片、Markdown 或 PDF 文件。

## 11. 开发者生态与安全 (Developer Ecosystem & Security)
*   **目标**：吸引二次开发，确保私有化部署安全。
*   **需求点**：
    *   **插件 SDK**：定义标准 Java 接口与注解，开发者只需编写单一类即可自动在前端注册新工具。
    *   **模型聚合接入**：原生支持 OpenAI、DeepSeek、Ollama 以及通过 OneAPI 协议接入的所有模型。
    *   **调试监控面板**：内置轻量级链路追踪 (Tracing)，展示 LLM 的原始 Prompt、调用耗时及命中上下文。
    *   **数据脱敏与安全**：支持 API Key 前端 LocalStorage 隔离存储，日志自动遮蔽敏感信息。
    *   **完善的贡献指南**：提供 `CONTRIBUTING.md`，明确说明如何贡献新 Agent 或新工具。
